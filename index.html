SVM-NAV: CONSTRAINT-AWARE VISION-LANGUAGE NAVIGATION WITH SUPERPIXEL VALUE MAP

Anonymous Author

Paper Supplementary Code

Abstract

We present SVM-Nav, a training-free approach for Vision-Language Navigation in Continuous Environments (VLN-CE). SVM-Nav reframes VLN-CE as a sequential sub-instruction completion problem, continuously translating sub-instructions into navigation plans via a cross-modal value map. Central to our approach are the Sub-Instruction Manager (SIM) and Superpixel Value Map (SVM). SIM decomposes instructions into sub-instructions and tracks navigation progress through automatic sub-instruction switching. In parallel, SVM generates a value map from vision-language model prompts based on landmarks identified by SIM, further refined through superpixel clustering to improve navigation stability. SVM-Nav achieves the state-of-the-art on two VLN-CE benchmarks, surpassing the second-best method by 12\% on R2R-CE and 13\% on RxR-CE in terms of Success Rate on the validation unseen split. Furthermore, SVM-Nav demonstrates robustness in real-world robot deployments across diverse indoor scenes and instructions, highlighting its potential for practical applications.

Method Overview

Overview of the proposed SVM-Nav. (a) The Sub-Instruction Manager decomposes the instruction into a sequence of sub-instructions and plans object constraints, location constraints and direction constraints for each of them. (b) During navigation, SVM-Nav builds a value map and uses the superpixel clustering method to segment it into regions. It automatically switches constraints and chooses the most promising region's geometric center as waypoints.

Framework of SVM-Nav

We present SVM-Nav, a training-free approach for VLN-CE. Within each episode, a Sub-Instruction Manager decomposes instructions and conducts constraint-aware sub-instruction switching. The active sub-instructions are translated into a Superpixel Value Map, which abstracts the environment and generates navigation plans. These plans are then executed by classical control algorithms, guiding the agent to complete each sub-instruction until the episode terminates.

Visualization in Simulation

Navigation on episode 1313 with Superpixel-based waypoint selection.

Instruction: Go straight. Pass the stairs on the right and continue straight. When you get to the stairs going up pass those as well. Go into the room with the couches and then turn right. wait near the glass table with white chairs.

SVM avoids focusing solely on local high values. Instead, it considers the global value map, allowing it to navigate correctly when switching sub-instructions.

Navigation on episode 1313 with frontier-based waypoint selection.

Instruction: Go straight. Pass the stairs on the right and continue straight. When you get to the stairs going up pass those as well. Go into the room with the couches and then turn right. wait near the glass table with white chairs.

The FBE-based waypoint selection method navigates correctly during the first sub-instruction. However, upon reaching the second sub-instruction, the landmark prompt changes to “room,” causing the value map, initially based on the previous landmark prompt “stairs”, to decay as new values are updated. Then the agent chooses a left frontier, which has the highest value among the available options, but the correct path is to walk straight toward the open area with the stairs.
